# SEO Content Writer — Kazkar.ai

## Purpose
Write blog articles that rank for target keywords while providing genuine value to TTRPG players and DMs.

## Inputs
- Content plan: `outputs/kazkar-seo-content-plan-YYYY-MM-DD.md`
- `knowledgebase/voice.md` — brand voice
- `knowledgebase/product.md` — product details (for natural mentions)
- Article brief from content plan (keyword, title, type, CTA type)

## Process

### Step 1: Research
1. Search for the target keyword
2. Read top 5 ranking articles
3. Identify: What do they cover? What do they miss?
4. Find the angle: What can we add that others don't?

### Step 2: Outline
Create a detailed outline:
- H1: Title (include primary keyword naturally)
- H2s: Main sections (4-6)
- H3s: Subsections where needed
- Key points for each section
- Where product mention fits (if CTA type allows)
- FAQ section (for featured snippet potential)

### Step 3: Write
Follow these rules:
- **Lead with value** — the article must be useful even if Kazkar didn't exist
- **TTRPG-native language** — write like a DM, not a marketer
- **Scannable** — short paragraphs, bullet points, headers
- **Practical** — include actionable tips, templates, examples
- **Primary keyword** in title, first paragraph, one H2, and naturally throughout
- **Related keywords** sprinkled in subheadings and body
- **Internal links** to other blog posts and relevant kazkar.ai pages
- **Meta description** — compelling, includes keyword, under 160 chars

### Step 4: SEO checklist
- [ ] Title under 60 characters
- [ ] Meta description under 160 characters
- [ ] Primary keyword in first 100 words
- [ ] At least one image suggestion with alt text
- [ ] Internal links to 2-3 other pages
- [ ] FAQ section with 3-5 questions
- [ ] No keyword stuffing (reads naturally)
- [ ] CTA matches the assigned type (none/soft/direct)

## Output
Save to `outputs/kazkar-seo-article-<slug>-YYYY-MM-DD.md`:
1. **Meta**: title, meta description, target keyword, slug
2. **Article**: full markdown text
3. **SEO notes**: internal links, image suggestions, schema markup recommendations
4. **Review status**: draft / reviewed / published
